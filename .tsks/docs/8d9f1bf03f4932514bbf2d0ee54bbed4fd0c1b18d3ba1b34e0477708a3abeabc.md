Add a boolean `--json` flag to the show command. When set, output the single task as a valid JSON object instead of the default human-readable format.

**Implementation details:**
- Add `--json` flag (bool, default false) to `cmd/show.go`
- When `--json` is true, serialize the task as a single JSON object using `encoding/json`
- Output must be **only** valid JSON — no headers, footers, or status messages to stdout
- Any errors should still go to stderr
- Should include the task detail file content in the output

**JSON schema:**
```json
{
  "id": "10",
  "title": "Machine-readable JSON output (--json flag)",
  "status": "todo",
  "created_at": "2026-04-07T11:59:14.075393011Z",
  "doc_hash": "c6f3f7dd551a9b92b05f796700e55c32f71b259defce32cdc1d76659c2d9619c",
  "dependencies": [],
  "detail": "Phase 1: Add --json flag to all commands..."
}
```
- Same field conventions as `list --json` (RFC 3339 timestamps, arrays not null)
- `detail` field contains the full markdown content of the task's detail file
- If no detail file exists, `detail` should be an empty string (`""`), not null

**Acceptance criteria:**
- `tssk show 10 --json | jq .` produces valid parsed JSON
- `tssk show 10 --json | jq '.title'` outputs just the title string
- No trailing whitespace or extra output on stdout
- Unit test verifies JSON validity and structure
