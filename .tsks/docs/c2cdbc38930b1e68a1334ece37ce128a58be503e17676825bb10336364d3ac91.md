Phase 3: Build comprehensive documentation site with versioned reference

## Phase 3 – v1.0

**Goal:** Production-ready v1.0 release with a stable public API, comprehensive documentation, and distribution packaging.

**Technical milestones:**
- Semantic versioning (`v1.0.0`) tag published
- Pre-built binaries for Linux (amd64/arm64), macOS (amd64/arm64), and Windows (amd64)
- All public commands and flags documented in a versioned reference

**Success metrics:**
- Zero open P0 bugs at GA
- Documentation covers 100% of public commands
- Binary size < 15 MB
- Install-to-first-task time < 2 minutes (measured against documented quick-start)

## Files to Modify/Create

### Modify
- `README.md` — Link to documentation site from the main README
- Existing `docs/` markdown files — May be used as source content for the docs site

### Create
- `docs/mkdocs.yml` or `hugo.toml` — MkDocs or Hugo configuration file
- `docs/site/` — Documentation site source files (if using MkDocs: `mkdocs.yml` + `docs/`; if Hugo: `hugo.toml` + `content/`)
- `.github/workflows/docs.yml` — CI workflow to build and deploy the documentation site (e.g., to GitHub Pages)
- `docs/site/docs/` — Markdown content files for the documentation site
- `docs/site/overrides/` — Theme customizations if needed
